+# Skill 01 ΓÇö Contracts Process
+
+## Skill name & intent
+
+**Name:** `contracts-process`
+
+**Intent:** Enable a Copilot Studio autonomous agent to understand, validate and monitor commercial **Contracts** between a `csp_BusinessEntity` and its counterparties (client accounts and contractors). The agent uses this skill to:
+
+1. Verify that contract records are internally consistent (required fields, valid status transitions, correct derivations).
+2. Detect discrepancies against business rules and downstream artefacts (invoices, timesheets, milestones, FX rates).
+3. Raise findings as `csp_CorporateAction` records so a human admin can review and act on them via the Corporate Actions board.
+
+This skill is **read-mostly**: the agent reads contract and related-entity data and writes only to `csp_CorporateAction`.
+
+---
+
+## In-scope Dataverse tables
+
+| Logical name | Role in the process | Key fields the agent reads |
+|---|---|---|
+| `csp_Contract` | Primary entity. The commercial agreement. | All fields below. |
+| `csp_BusinessEntity` | The legal entity (CSP-RO / CSP-BG / CSP-US) issuing the contract. | `csp_name`, `csp_country`, `csp_baseCurrencyCode` |
+| `csp_Account` | Parent (client/payer) and optional child account on the contract. | `csp_accountNumber`, `csp_name`, `csp_status`, `csp_paymentTerms` |
+| `csp_Contact` | The contractor / assignee. | `csp_firstName`, `csp_lastName`, `csp_contactType`, `csp_availableForWork` |
+| `csp_ExchangeRate` | FX rates used when sell and buy currencies differ. | `csp_fromCurrencyCode`, `csp_toCurrencyCode`, `csp_rate`, `csp_effectiveDate` |
+| `csp_Invoice` | Billing artefacts referencing a contract. | `csp_invoiceNumber`, `csp_status`, `csp_periodMonth`, `csp_periodYear`, `csp_contractId` |

+## Lifecycle & status transitions
+
+```
+Draft ΓöÇΓöÇΓû╢ Active ΓöÇΓöÇΓû╢ On Hold ΓöÇΓöÇΓû╢ Active
+                Γöé                   Γöé
+                Γö£ΓöÇΓöÇΓû╢ Completed     Γöé
+                ΓööΓöÇΓöÇΓû╢ Terminated ΓùÇΓöÇΓöÇΓöÿ
+```
+
+Allowed transitions:
+
+| From | To | Preconditions |
+|---|---|---|
+| `Draft` | `Active` | All required fields populated; counterparty accounts/contact `Active`. |
+| `Active` | `On Hold` | None. |
+| `On Hold` | `Active` | None. |
+| `Active` / `On Hold` | `Completed` | `csp_actualEndDate` set; no pending milestones in `Pending`. |
+| `Active` / `On Hold` | `Terminated` | `csp_actualEndDate` set. |

+
+- Only `Active` contracts feed timesheet generation and invoice creation.
+- `Completed` / `Terminated` contracts must not appear in current-month forecasts.

+## Validation rules (agent-runnable checks)
+
+For each rule: **Given ΓåÆ Assert ΓåÆ Else raise** discrepancy `D-NN` (see catalog).
+
+1. **R1 ΓÇö Required fields.** Given a contract with `statuscode != Draft`, assert all required fields are populated. Else raise `D-01`.
+2. **R2 ΓÇö Date ordering.** Assert `csp_endDate >= csp_startDate` when `csp_endDate` is set, and `csp_actualEndDate >= csp_startDate` when set. Else raise `D-02`.
+3. **R3 ΓÇö Distinct accounts.** Assert `csp_parentAccountId != csp_childAccountId`. Else raise `D-03` (High).
+4. **R4 ΓÇö Fixed Price configuration.** Given `csp_contractType = Fixed Price` AND `csp_billingType = Fixed Price`, assert `csp_hasTimesheet = false` AND `csp_hasMilestones = true`. Else raise `D-04` (High).
+5. **R5 ΓÇö Permanent employee salary.** Given `csp_contractType = Permanent Employee`, assert `csp_monthlySalary > 0` AND `csp_monthlySalaryCurrency` set. Else raise `D-05` (High).
+6. **R6 ΓÇö Termination requires actual end date.** Given `statuscode = Terminated`, assert `csp_actualEndDate` is set. Else raise `D-06` (High).
+7. **R7 ΓÇö Active past end date.** Given `statuscode = Active` AND `csp_endDate < today`, raise `D-07` (High).
+8. **R8 ΓÇö Margin floor.** Given derived `csp_marginPercent < 10`, raise `D-08` (Medium).
+9. **R9 ΓÇö Currency mismatch with stale FX.** Given `csp_sellCurrency != csp_buyCurrency` AND latest matching `csp_ExchangeRate.csp_effectiveDate` is older than 30 days (or missing), raise `D-09` (Low).
+10. **R10 ΓÇö Counterparty status.** Given the parent account or contact is `Inactive`, raise `D-10` (Medium).
+11. **R11 ΓÇö Timesheet expected but missing.** Given `statuscode = Active` AND `csp_hasTimesheet = true` AND no `csp_Timesheet` exists for the contract for the previous calendar month, raise `D-11` (Medium).
+12. **R12 ΓÇö Milestone overdue.** Given a related `csp_ContractMilestone` with `csp_status = Pending` AND `csp_endDate < today`, raise `D-12` (Medium).
+13. **R13 ΓÇö Mutually exclusive flags.** Given `csp_hasTimesheet = true` AND `csp_hasMilestones = true`, raise `D-13` (High) ΓÇö these must not both be true.
+14. **R14 ΓÇö Account payment-terms drift.** Given `csp_payTerms` differs from the parent account's current `csp_paymentTerms`, raise `D-14` (Low).
+

+## Derivations & formulas
+
+- **Hourly from daily:** if `csp_unitOfMeasure = Day`, then `csp_sellHourlyRate = csp_sellRate / 8` and `csp_buyHourlyRate = csp_buyRate / 8`.
+- **Same-currency margin:** `csp_margin = csp_sellRate ΓêÆ csp_buyRate`; `csp_marginPercent = csp_margin / csp_sellRate ├ù 100`.
+- **Cross-currency margin (display):** convert `csp_buyRate` from `csp_buyCurrency` ΓåÆ `csp_sellCurrency` using the most recent `csp_ExchangeRate` whose `csp_effectiveDate <= today`. Then apply the same-currency formula. Flag the result as `display-only` because the booked margin uses period-specific rates.
+- **Invoice line amount suggestion:** `quantity ├ù (csp_sellRate if UoM = invoice line UoM, else csp_sellHourlyRate when invoicing in Hours)`.
+
+---
+
+## Cross-entity effects & integrity
+
+- **Timesheet eligibility:** only contracts with `statuscode = Active` AND `csp_hasTimesheet = true` AND NOT (`csp_contractType = Fixed Price` AND `csp_billingType = Fixed Price`) appear in the timesheet generator.
+- **Milestone eligibility:** only contracts with `csp_hasMilestones = true` are eligible for milestone records.
+- **Deletion blockers:** a contract cannot be deleted if any `csp_Invoice`, `csp_Expense`, `csp_Timesheet`, or `csp_ContractMilestone` references it.
+- **Account / contact dependencies:** if the contractor (`csp_contactId`) or parent account becomes `Inactive`, surface a discrepancy (R10) but do not auto-deactivate the contract.
+

+## Discrepancy catalog (write to `csp_CorporateAction`)
+
+For each discrepancy the agent creates one `csp_CorporateAction` row. Use the following defaults:
+
+| ID | Title template | Priority | Due-date offset | `csp_ActionDetails` body (template) |
+|---|---|---|---|---|

+| D-04 | `Fixed Price contract {csp_contractNumber} misconfigured` | High | +2 days | Explain that `csp_hasTimesheet` must be false and `csp_hasMilestones` must be true for Fixed Price + Fixed Price. |
+| D-05 | `Permanent Employee contract {csp_contractNumber} missing monthly salary` | High | +2 days | Show contact name and ask admin to set `csp_monthlySalary` and `csp_monthlySalaryCurrency`. |
+| D-06 | `Terminated contract {csp_contractNumber} missing actual end date` | High | +1 day | Ask admin to set `csp_actualEndDate`. |
+| D-07 | `Active contract {csp_contractNumber} past planned end date` | High | +3 days | Show `csp_endDate`, days overdue, and ask whether to extend, complete, or terminate. |
+| D-08 | `Contract {csp_contractNumber} margin below 10%` | Medium | +7 days | Show derived `csp_marginPercent`, sell/buy rates and currencies. |
+| D-09 | `Contract {csp_contractNumber} uses stale FX rate` | Low | +14 days | Show currency pair, last `csp_ExchangeRate.csp_effectiveDate`, and the staleness in days. |
+| D-10 | `Contract {csp_contractNumber} references inactive counterparty` | Medium | +7 days | Identify which of parent account / contact is inactive. |
+| D-11 | `Active contract {csp_contractNumber} has no timesheet for {previousMonth}` | Medium | end of current month | List contractor, contract period, and the missing month. |
+| D-12 | `Pending milestone overdue on contract {csp_contractNumber}` | Medium | +3 days | Show milestone description, planned end, days overdue. |
+| D-13 | `Contract {csp_contractNumber} has both timesheet and milestone flags enabled` | High | +2 days | Ask admin to choose one billing model. |
+| D-14 | `Contract {csp_contractNumber} payment terms drifted from account` | Low | +14 days | Show contract `csp_payTerms` vs account `csp_paymentTerms`. |

+3. **R3 ΓÇö Mutual exclusion.** A line cannot have both matches. Else `D-03` (High).
+4. **R4 ΓÇö Amount tolerance.** Matched invoice/expense amount within ┬▒0.01 of line value. Else `D-04` (Medium).
+5. **R5 ΓÇö Currency match.** Statement currency = matched record currency. Else `D-05` (Medium).

+7. **R7 ΓÇö Negative margin.** Per applicant: hourly margin (= header rate/h ΓêÆ applicant rate/h, with dayΓåöhour ├ù8 conversion) `< 0`. Raise `D-07` (Medium).
+8. **R8 ΓÇö Closing date past + still open.** `csp_ClosingDate < today` and `statuscode Γêê {New, Interview Booked}`. Raise `D-08` (Medium).
+9. **R9 ΓÇö Won/Lost without outcome comment.** `statuscode = Won` or `Lost` with empty `csp_OutcomeComments` after 3 days. Raise `D-09` (Low).
+10. **R10 ΓÇö Stuck Drafted.** Applicant `statuscode = Drafted` for >7 days on a non-`Lost` opportunity. Raise `D-10` (Medium).
+11. **R11 ΓÇö Sent without recipient.** Send action invoked but neither account nor prospect's primary contact email resolves and `csp_NewClientName` is not an email. Raise `D-11` (High).
+12. **R12 ΓÇö Profile shared outside an opportunity.** Any send-profile event referencing a `csp_Contact` without a corresponding `csp_OpportunityApplicant` row. Raise `D-12` (High).
+13. **R13 ΓÇö Recruiter-sourced candidate applied directly.** Applicant `csp_Candidate.csp_source = Recruiter` and Send happened to a non-recruiter recipient. Raise `D-13` (High).

+# Skill 17 ΓÇö Reports (Billing & Profit) Process
+
+## Skill name & intent
+
+**Name:** `reports-process`
+
+**Intent:** Govern the forward-looking financial dashboard: actuals for last/this month and forecast for next 2 months, broken down by country and currency. The agent does **not write** to financial tables; it validates inputs are present and consistent so the report renders correctly, and raises Corporate Actions when forecast inputs (FX, holidays, contract dates, milestones) are missing.
+
+---
+
+## In-scope Dataverse tables
+
+| Logical name | Role |
+|---|---|
+| `csp_Contract` | Forecast input. |
+| `csp_ContractMilestone` | Fixed-Price month allocation. |
+| `csp_Invoice` / `csp_InvoiceLine` | Last-month profit override. |
+| `csp_Expense` | Last-month profit override (Contractor Payments). |
+| `csp_ExchangeRate` | Cross-currency normalisation. |
+| `csp_PublicHoliday` | Working-day calculation per entity country. |
+| `csp_ReportingSnapshot` | Optional persisted snapshots. |
+
+---
+
+## Computed columns (per month bucket: `last`, `this`, `next`, `inTwoMonths`)
+
+- **Billing forecast (per contract):**
+  - `Time & Material` / `Standard Contracting` (Day): `sellRate ├ù workingDaysInMonth ├ù allocation`.
+  - `Time & Material` (Hour): `sellHourlyRate ├ù workingDaysInMonth ├ù 8`.
+  - `Monthly Salary`: `sellRate` (treated as monthly).
+  - `Fixed Price`: pro-rate `csp_grossValue` over duration months OR include a milestone falling in that month.
+- **Profit forecast (per contract):** `billing ΓêÆ cost`, cost via `csp_buyRate`. Cross-currency uses latest `csp_ExchangeRate`.
+- **Working days** = weekdays in month minus matching `csp_PublicHoliday` for the contract's entity country.
+
+### Last-month override
+
+For the `last` bucket, **profit** is replaced with actual profit derived from `csp_InvoiceLine` linked to the contract minus matched `csp_Expense` (`Contractor Payment`) linked to the contract.
+
+---
+
+## Aggregation
+
+- Headline grouping: `(country, currency)`, 4 months ├ù `(billing, profit)`.
+- Drill-down: per-contract rows under each headline group.
+- Single chart: 4 months ├ù `(billing, profit)`, summed in display currency (FX warning when mixing).
+
+---
+
+## Validation rules (read-only ΓÇö emits discrepancies, no writes to financials)
+
+1. **R1 ΓÇö Missing FX rate for forecast period.** Contract has `csp_sellCurrency Γëá csp_buyCurrency` and no `csp_ExchangeRate` for the forecast month. Raise `D-01` (Medium).
+2. **R2 ΓÇö Stale FX rate.** Latest matching rate older than 30 days. Raise `D-02` (Low).
+3. **R3 ΓÇö Missing public holidays.** Entity country has < 1 `csp_PublicHoliday` for the forecast year. Raise `D-03` (Medium).
+4. **R4 ΓÇö Active contract missing dates.** `csp_startDate` null or invalid. Raise `D-04` (High).
+5. **R5 ΓÇö Fixed Price without gross value or milestones.** Contract `Fixed Price` with `csp_grossValue = 0` and no milestones in any forecast month. Raise `D-05` (Medium).
+6. **R6 ΓÇö Last-month profit anomaly.** Override profit deviates from forecast by >50%. Raise `D-06` (Low) ΓÇö informational variance.
+7. **R7 ΓÇö Snapshot drift.** Persisted `csp_ReportingSnapshot` differs from recomputed value by more than tolerance. Raise `D-07` (Low).
+
+---
+
+## Permissions
+
+Admin and Owner only.

+|---|---|---|---|
+| D-01 | `FX rate missing for {currencyPair} in {month}` | Medium | +5d |

+# Business Skills

+This folder contains **Dataverse Business Skills** ΓÇö declarative process specifications written for **Copilot Studio autonomous agents**.
